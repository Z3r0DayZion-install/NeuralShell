Institutional Evaluation Guide

Use this page to run a structured 30-minute evaluation from verified product capabilities. All claims are testable.

Integrity notice: No certification or compliance claim should be made unless independently earned and documented. Verify before asserting.

What NeuralShell Is

  • Local-first AI operator shell with hardware-bound sessions
  • Offline-capable with air-gapped model execution
  • Evidence-first continuity with audit export
  • Verifiable — all claims backed by reproducible proof

What NeuralShell Is Not

  • Cloud-only monitoring or observability product
  • Offensive security tooling
  • Auto-generated compliance guarantee system
  • Replacement for your existing SIEM or SOC tooling

Evaluation Success Criteria

These are the three gates that should pass before recommending a pilot.

1
Demo path executes without manual repair
Install the app, run the hardware-binding proof flow, and confirm session restore works end-to-end without intervention. See proof page for all 7 scenarios.
2
Security and procurement pack generated from runtime truth
Run .\VERIFY_RELEASE.ps1 to confirm installer hash. Review SOC2_PREP_REPORT.md for compliance posture. Audit artifacts are in proof/latest/.
3
Pilot scope and evidence criteria agreed in writing
Review the 14-day pilot scope document before committing. Pilot should have a defined baseline, a guided walkthrough, tracked metrics, and a final recommendation deliverable.

30-Minute Walkthrough Order

Run these in sequence. Each step has a pass/fail outcome.

1
Download and verify installer (5 min)
Download from GitHub Releases. Run .\VERIFY_RELEASE.ps1. Confirm hash matches SHA256SUMS.txt.
2
Install and first launch (5 min)
Run installer, confirm hardware identity is established, connect a model provider (Ollama for offline, API key for cloud).
3
Run hardware-binding proof flow (10 min)
Create thread, export session, wipe app data, restore — confirm session recovers on original machine. Attempt restore on a second machine to verify failure.
4
Validate offline operation (5 min)
Disconnect network, confirm local model execution continues with no errors or cloud fallback.
5
Review audit export (5 min)
Generate logs and validation artifacts. Confirm they are structured, readable, and sufficient for your compliance or security team to review independently.
