# PROCUREMENT_FAQ

## Q: Does NeuralShell require cloud access for core operations?

A: No. Core runtime workflows operate local-first and support air-gapped environments.

## Q: How is trust identity handled?

A: Through local certificate authority controls with issuance, rotation, and revocation lifecycle management.

## Q: How are artifacts moved between restricted environments?

A: Via signed offline courier packages with quarantine, receipt verification, and release controls.

## Q: Can continuity posture be demonstrated?

A: Yes. Continuity drills generate run evidence, state deltas, and readiness scores.

## Q: Can security review docs be regenerated for new versions?

A: Yes. Procurement/security pack generators produce versioned outputs with delta comparison.
