# Pilot Rollout Strategy: Verification Signal

**Current Stage:** Verification Pilot for a Deterministic Desktop Runtime.

## 1. Goal
Collect data on the viability of the "Verify-then-Run" distribution model. 
- Will people actually verify software?
- Does SmartScreen kill trust?
- is the complexity manageable?

## 2. Target Demographic
- **Quantity:** 20–30 technically competent testers.
- **Candidates:**
    - Self-hosting/Privacy communities.
    - Security engineers.
    - Developers familiar with hash verification.
    - Power users comfortable with PowerShell/CLI.

## 3. Deployment Protocol
1. **Selection:** Identify and vet ~30 candidates.
2. **Invitation:** Send the *Pilot Invite Email*.
3. **Distribution:** Upon interest, provide the *Distribution ZIP* + *Instructions* + *Feedback Form*.
4. **Collection:** Log all feedback into the `governance/PILOT_LEDGER.jsonl` (to be created).
5. **Analysis:** Analyze signal for the "Trust Boundary" between the user and the software.

## 4. Warning
Do not market this as a finished product. Frame it strictly as a **verification pilot**.
