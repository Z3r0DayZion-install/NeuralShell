# Backlog Operations Runbook

## Purpose
Operate the maintenance backlog as a decision board, not a dumping ground.

## Intake Rules
A backlog item must include:
- reproducible symptom
- observed impact
- evidence source
- proposed state
- owner

## Review Cadence
- Weekly: triage new items and state changes
- Monthly: confirm priorities, close stale observation items, and re-validate deferred items

## Board Flow
1. Intake and evidence validation
2. State assignment using `BACKLOG_STATE_MODEL.md`
3. Owner and due date assignment
4. Weekly state review
5. Closure with verification note

## Closure Rules
Close only when:
- fix or disposition is documented
- evidence confirms expected behavior
- no unresolved dependency blocks closure
