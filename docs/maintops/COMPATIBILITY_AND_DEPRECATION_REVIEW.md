# Compatibility and Deprecation Review

## Purpose
Review compatibility drift and deprecation decisions under maintenance policy.

## Review Cadence
- Monthly in normal operations
- Immediate review when a high-impact compatibility signal appears

## Compatibility Review
Check for:
- runtime/toolchain drift
- dependency support changes
- platform behavior changes affecting reproducibility
- documentation mismatches with current behavior

## Deprecation Decision Rules
A deprecation action requires:
- clear rationale
- impact assessment
- migration note
- communication plan and timing

## Communication Threshold
Communication is required when:
- operator workflow changes
- support mode boundaries affect expected behavior
- compatibility constraints change user setup requirements

## Output
- compatibility findings
- deprecation decisions
- communication actions and owners
