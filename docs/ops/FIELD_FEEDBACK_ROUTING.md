# FIELD_FEEDBACK_ROUTING

## Goal

Capture field feedback in structured form and route it into the correct product/support/docs/training queues.

## Classification Categories

- Product gap.
- Deployment friction.
- Buyer confusion.
- Support issue.
- Docs/training gap.
- Packaging/pricing confusion.

## Routing Outputs

- Per-note category and route assignment.
- Severity tagging.
- Weekly route count summary.
- High-severity blocker highlights with account context.

## Safety Rule

Feedback summaries must reflect captured notes only; avoid synthetic metrics or fabricated outcomes.
