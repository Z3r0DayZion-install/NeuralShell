# NeuralShell V2.1.8 — Maintenance Release Criteria

## 1. Release Triggers
A maintenance release is justified when:
1. One or more verified P0/P1 patches are ready in the pipeline.
2. A critical compatibility boundary drift is identified as per `COMPATIBILITY_BOUNDARIES.md`.
3. Cumulative P2/P3 items reach a volume that impacts overall stewardship efficacy.

## 2. Exclusion Rules
- Zero releases for cosmetic-only changes.
- Zero releases for documentation-only changes.
- Zero releases for unverified findings.

## 3. Versioning
Maintenance releases follow the V2.1.x line (e.g., V2.1.4 -> V2.1.5-maint). The "Golden Master" label is preserved as the foundation.
