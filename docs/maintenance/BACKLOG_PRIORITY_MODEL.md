# NeuralShell V2.1.8 — Backlog Priority Model

## 1. P0 — Critical Maintenance
- **Trigger**: security regression, total startup failure, OMEGA seal breakage.
- **Action**: Activate `PATCH_PIPELINE.md` immediately.
- **Goal**: Resolve within 48h.

## 2. P1 — Major Maintenance
- **Trigger**: Primary feature regression with no workaround.
- **Action**: Target for the next cumulative Maintenance Release.
- **Goal**: Resolve within 14 days.

## 3. P2 — Minor Maintenance
- **Trigger**: Feature regression with an established workaround.
- **Action**: Deferred to the next minor baseline or future roadmap.
- **Goal**: Review monthly for severity drift.

## 4. P3 — Trivial Maintenance
- **Trigger**: Cosmetic jitter, doc typo, telemetry noise.
- **Action**: Support-only or deferred Indefinitely.
